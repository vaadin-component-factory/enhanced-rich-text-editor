## Design Decisions

- **Single guide instead of 4 docs:** Originally planned User Guide + Developer Guide + API Reference + Upgrade Guide. Consolidated to TABLES_GUIDE.md + TABLES_UPGRADE_GUIDE.md — less navigation for developers, no content duplication.
- **No screenshots:** Tables explored in playground, docs focus on code-level API. Screenshots would be outdated quickly.
- **Human-readable style:** Written conversationally per user request, with practical examples and "what you get" framing.
