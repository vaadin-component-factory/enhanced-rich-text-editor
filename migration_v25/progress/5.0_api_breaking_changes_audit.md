# Phase 5.0: Public API Breaking Changes Audit

**Status:** NOT STARTED

## Goal

Complete code review of all public API changes between ERTE V24 (1.x) and ERTE V25 (6.x), including the Tables addon. Produce a simple, actionable list of every removed, renamed, or signature-changed public method/class/constant. For each entry, assess whether the breaking change is **necessary** (e.g., Jackson 3 migration, Vaadin 25 API change) or **avoidable** (removed without compelling reason — candidate for restoration).

## Scope

### Core ERTE
- V24: `enhanced-rich-text-editor/src/main/java/com/vaadin/componentfactory/`
- V25: `enhanced-rich-text-editor-v25/src/main/java/com/vaadin/componentfactory/`

### Tables Addon
- V24: `enhanced-rich-text-editor-tables/src/main/java/com/vaadin/componentfactory/erte/tables/`
- V25: `enhanced-rich-text-editor-tables-v25/src/main/java/com/vaadin/componentfactory/erte/tables/`

### Existing Reference
- Upgrade Guide: `enhanced-rich-text-editor-v25/UPGRADE_GUIDE.md` (already documents some changes)
- Phase 4.3 progress file Issue 2 references this audit

## Deliverable

A simple list in this file, structured as:

```
### [ClassName]
| V24 Signature | V25 Status | Necessary? | Notes |
|---|---|---|---|
| `method(JsonObject)` | Changed to `method(ObjectNode)` | Yes — Jackson 3 | |
| `getSomething()` | REMOVED | No — restore | Was used by X |
```

## Method

1. List all public/protected classes in V24
2. For each class, list all public/protected methods
3. Compare with V25 equivalent
4. Categorize: kept, signature-changed (necessary), removed (assess), added (new)
5. Cross-reference with Upgrade Guide — flag undocumented changes
6. Output: the list above, plus recommendations

## Out of Scope
- Internal/private API changes
- JS/client-side API (separate audit if needed)
- Test classes
