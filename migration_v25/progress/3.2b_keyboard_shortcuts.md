# Phase 3.2b: Custom Keyboard Shortcuts (Feature 10)

## Status: COMPLETE

## Feature Summary
`addStandardToolbarButtonShortcut()` binds key combos to toolbar button actions.
`addToolbarFocusShortcut()` binds key combo to focus toolbar.

## Implementation

### JS Methods (vcf-enhanced-rich-text-editor.js)
- `addStandardToolbarButtonShortcut(partSuffix, key, shortKey, shiftKey, altKey)` — finds button by part suffix, prepends keyboard binding that `.click()`s the button
- `addToolbarFocusShortcut(key, shortKey, shiftKey, altKey)` — prepends binding that calls `_markToolbarFocused()` and focuses first visible button
- Both use SHORTKEY pattern (metaKey on Mac, ctrlKey elsewhere)
- Both prepend to `keyboard.bindings[key]` array for priority

### Java API (EnhancedRichTextEditor.java)
- `addStandardToolbarButtonShortcut(ToolbarButton, String key, boolean shortKey, boolean shiftKey, boolean altKey)`
- `addToolbarFocusShortcut(String key, boolean shortKey, boolean shiftKey, boolean altKey)`
- Direct `executeJs()` calls (one-shot registrations, not cached state)

### V24 → V25 Changes
- `Number keyCode` → `String key` (Quill 2 string key names)
- `Boolean` wrappers → `boolean` primitives
- Fixed typo: `addToobarFocusShortcut` → `addToolbarFocusShortcut`
- `.click()` strategy instead of V24's format-name handler mapping
